Catalog of forums, communities, tools, and file-format research drawn on for this whole modding
section — about *where the community keeps its actual technical knowledge*, as opposed to
[mods survey](./mods-survey.md) which lists individual mods. Surveyed 2026-07-09.

**Status legend**: ✅ = opened and read directly · ⬜ = cited/referenced only — found via a search
result, a cross-link, or carried over from an earlier pass, but never actually opened. Being listed
here is not the same as being verified; the tag says which is which.

## Key technical facts confirmed here
- **Archive format versioning**: Far Cry 2 uses **FAT version `-v5`** (paired `.dat`/`.fat` files). Far Cry 3/4 use `-v9`, Far Cry 5 uses `-v10` — same lineage, different version per game, per ZenHAX's Dunia archive research.
- **Files are stored as hashes, not names.** The `.dat`/`.fat` archives contain no filename strings — only hashes — so a maintained hash→filename list is required to make sense of an unpacked archive. This is exactly why a "Hash Decoder" exists as a separate tool (see Enhanced Gibbed Tools below).
- **Data is authored/edited as XML.** The existence of a dedicated "FC2 XML Decoder" (used by SCHTEVE) confirms the game's data files round-trip through an XML representation for editing, even though the packed format on disk isn't plain XML.
- **Textures need a dedicated converter** (`xbt2dds`) — the game's texture format isn't directly DDS and must be converted both ways.
- **FCB** is a distinct binary format (see FCBConverter below) used across the Far Cry series, separate from the `.dat`/`.fat` archive container — likely the per-file binary format *inside* the archives (config/entity data).

## Tools (beyond what's already in the mods survey)

- **[FC2-Dunia-Fixes](https://github.com/seamusduncmcgrath/FC2-Dunia-Fixes)** ⬜ - A comprehensive ASI plugin for Far Cry 2 that overhauls the engine's legacy memory management, optimizes CPU multithreading, fixes long-standing graphical bugs, and adds modern quality-of-life features.

No README (GitHub shows the zlib LICENSE instead) — 79 commits, still getting occasional work (latest: "Add initial Far Cry Primal support"), 2 contributors: Gibbed himself and **Janne252** — the same person behind "Janne252's editor mod" on fc2mp.com, a useful cross-reference confirming that's an actively-contributing community member, not just a mod author. - **[Enhanced Gibbed Tools with Hash Decoder (ModDB)](https://www.moddb.com/games/far-cry-2/downloads/enhanced-gibbed-tools-with-hash-decoder)** ⬜ — not directly opened; modified Gibbed tools bundling a hash decoder, directly solving the "files are hashes, not names" problem noted above, per its listing.

- **[FarCry2-Schteve (GitHub, tylerdotrar/FarCry2-Schteve)](https://github.com/tylerdotrar/FarCry2-Schteve)** ✅ — source for the SCHTEVE Nexus mod; README confirms it wraps **Gibbed.Dunia** (unmodified) + **xbt2dds** (slightly modified) + a **"FC2 XML Decoder"** (batch→PowerShell rewrite, original author unknown) behind a PowerShell TUI ("SCHTEVE" = "Scripted Helper Tool (for FC2) DEVElopers"). Config-driven "Sandbox" working directory model, color-coded menu showing what's set up. Actively maintained enough to have a demo video for v2.4.5.
- **[FarCry2_trainer (GitHub, GregoryKimball/FarCry2_trainer)](https://github.com/GregoryKimball/FarCry2_trainer)** ✅ — WPF app for tweaking/modding FC2, built on Gibbed's binaries (bundled, not just referenced). Single release, v1.0.0, 2018-04-28; tested only against Steam v1.03.
- **[Queiroz-Far-Cry-2-Modding-Tool (GitHub, Hoklifter/Queiroz-Far-Cry-2-Modding-Tool)](https://github.com/Hoklifter/Queiroz-Far-Cry-2-Modding-Tool)** ✅ — Python-based XML-modding facilitation tool (not C#/PowerShell like the others), README is just an install guide (pip + a `Run Modding Tool.py` entry point) with no feature list; links out to "FAR CRY MODDING GUIDE" and a "FAR CRY MODDING DISCORD COMMUNITY" (likely the same Discord already in this file, not independently verified — link itself not followed ⬜). Originally by GitHub user **Queiroz**, championed/tested on Windows by Discord user **hoklifter** starting late 2024 (Discord, `🔩-tools-talking`, 2024-11-21: *"Guys, i could not test this tool on windows, i use linux. Worth grabbing directly if the GitHub repo's HEAD ever proves less stable than a known-good pinned build. **Gibbed's build server is split into separate `dunia1`/`dunia2` build lineages** ⬜ (not directly confirmed for FC2, but strongly implied): a user struggling with three different unpackers was pointed to `http://svn.gib.me/builds/dunia2/` with *"just use dunia2, not dunia1"* (Discord, `🔨-fc3-and-bd-modding`, 2025-04-02, from fdx4061) — since FC2 predates FC3's "Dunia 2," a `dunia1` build at the same server would plausibly be the FC2-generation toolchain. Worth directly checking `svn.gib.me` for a `dunia1` tree if the pinned `dunia-r65_b10.zip` build above ever needs a newer alternative.
- **`Gibbed.Map.Unpack`** ⬜ (mentioned, not directly explored) — a map-specific variant/mode of the Gibbed toolset for unpacking custom multiplayer map files specifically (distinct from the general FCB/archive tools already documented) — per Discord `map-editor` (Dec 2025), it leaves some files labeled "unknown" due to gaps in its internal file-name list; Gabor maintains his own more-complete unpacker/repacker plus file-list for this specific format (see [Engine Theory](./engine-theory.md), custom map file structure).
